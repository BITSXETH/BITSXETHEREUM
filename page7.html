Front-End For AlphaCoin
Introduction

Frontend development in DApps involves creating the user interface (UI) and user experience (UX) components that interact with the decentralized backend. The frontend interacts with the decentralized backend through various protocols and APIs.
Frontend development in DApps offers unique challenges and opportunities compared to traditional web development. By understanding the decentralized architecture, leveraging appropriate tools and libraries, and prioritizing security and usability, frontend developers can create compelling user experiences for decentralized applications.


Prerequisites

Frontend development for most DApps is done through JavaScript using Node.js and assistant libraries like Web3.js and ether.js.

Web3.js is a JavaScript library that allows interaction with Ethereum-compatible blockchains. It enables frontend developers to connect their applications to the blockchain, facilitating actions like reading data from the blockchain, sending transactions, and listening to events. Other platforms have similar libraries, such as Eosjs for EOS and TronWeb for Tron.


Wallet Integration

DApps often require users to sign transactions using their cryptocurrency wallets. Integrating wallet functionalities into the frontend is crucial. Wallets like MetaMask provide browser extensions that can be integrated into DApp interfaces, allowing users to interact securely with the blockchain.


Front-End for Alphacoin

As an example, you are provided with the frontend code for the above mentioned “AlphaCoin”. This code is a very basic HTML page that serves as a simple interface for interacting with a smart contract on the Ethereum blockchain. The smart contract is a standard ERC-20 token contract, and the interface allows users to connect their Ethereum wallets, check their token balance, transfer tokens, and approve token allowances.

Let's break down the code section by section:

HTML Structure
 DOCTYPE and HTML tags: These define the document type and language.
Head: Contains metadata and includes the web3.js library.
Body: Contains the interface elements and scripts.

JavaScript Functions
 init()
This function initializes the web3 provider and checks if the user has an Ethereum-enabled browser extension (like MetaMask). If so, it prompts the user to connect their wallet and fetches the connected account's address. It then initializes the smart contract.
initContract()
This function initializes the smart contract by specifying its address and ABI (Application Binary Interface). ABI defines how to interact with the contract. The ABI includes several functions like balanceOf, transfer, approve, etc., which correspond to the ERC-20 standard.
updateBalance()
This function updates the displayed balance for the connected account by calling the balanceOf function from the smart contract.
transfer()
This function allows the user to transfer tokens to another Ethereum address. It retrieves the recipient's address and the amount to be transferred from the input fields and calls the transfer function of the smart contract.
approve()
This function approves a spender to spend tokens on behalf of the connected account up to a specified amount. It retrieves the spender's address and the allowance amount from the input fields and calls the approve function of the smart contract.
connectWallet()
This function simply calls the init() function to connect the user's Ethereum wallet when the "Connect Wallet" button is clicked.

Event Listeners
 window.addEventListener('load', async () => { ... });: This event listener triggers the init() function when the page is fully loaded, ensuring the web3 provider is initialized.

Web3.js Functions
Web3(window.ethereum): Initializes the web3 object with the Ethereum provider.
window.ethereum.enable(): Requests permission to access the user's Ethereum accounts.
web3.eth.getAccounts(): Fetches the Ethereum accounts available in the user's wallet.
new web3.eth.Contract(abi, address): Creates a contract instance using the ABI and contract address.
contract.methods.methodName().call(): Calls a view or pure method from the smart contract.
contract.methods.methodName().send({ from: account }): Sends a transaction to the smart contract.
